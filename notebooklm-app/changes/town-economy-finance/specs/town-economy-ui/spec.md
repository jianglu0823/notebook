# Spec: town-economy-ui

小镇视图(`#worldView`)的样式优化与经济数据可视化面板。纯前端(`index.html`),消费后端只读 API,不引入新的写操作。

## ADDED Requirements

### Requirement: 经济面板展示小镇整体金币与收支
小镇视图 SHALL 新增一个经济面板,展示小镇整体金币总量(`totalCoins`)、当日总收入、当日总支出与场所营收总额,数据来自经济快照只读 API。当接口无数据时 MUST 显示占位/空态而非报错。

#### Scenario: 有数据时展示汇总
- WHEN 经济快照 API 返回当日 `totalCoins` 与收支
- THEN 面板显示小镇金币总量与当日收入/支出/营收总额

#### Scenario: 无数据时空态
- WHEN 经济快照 API 返回空(尚无结算)
- THEN 面板显示空态提示,页面不报错

### Requirement: 展示居民收入与富豪榜
小镇视图 SHALL 展示居民经济数据:至少含按 `coins` 或净资产排序的富豪榜,以及点击某居民可见其日/月净收入与收入明细。

#### Scenario: 富豪榜排序
- WHEN 经济数据加载完成
- THEN 富豪榜按金币/净资产从高到低排列居民

#### Scenario: 查看居民收入明细
- WHEN 用户点击某居民查看收入
- THEN 展示该居民的日/月净收入及其构成明细(来自 resident-income API)

### Requirement: 展示场所营收
小镇视图 SHALL 展示各商业场所(食肆/杂货铺/回春医馆/集市等)的当日营收。

#### Scenario: 场所营收列表
- WHEN 场所营收 API 返回当日数据
- THEN 面板按场所列出各自当日营收金额

### Requirement: 展示股票行情与居民持仓
小镇视图 SHALL 展示股票行情(每只股票当前价与涨跌,近期走势)与可选的居民持仓查看。行情涨跌 SHALL 有视觉区分(涨/跌不同颜色)。

#### Scenario: 行情涨跌着色
- WHEN 某股票当日上涨、另一只下跌
- THEN 上涨项与下跌项以不同颜色区分显示

#### Scenario: 查看居民持仓
- WHEN 用户查看某居民的持仓
- THEN 展示其持有的股票、数量与浮动盈亏

### Requirement: 小镇视图样式优化
小镇视图 SHALL 在不破坏既有地图/居民/活动流功能的前提下优化视觉呈现(布局、层次、经济面板与地图协调)。既有的地图渲染、居民列表、活动流 MUST 在改版后仍可正常显示与交互。

#### Scenario: 改版不回归既有功能
- WHEN 进入优化后的小镇视图
- THEN 地图、居民列表、活动流、日报等既有模块仍正常展示与可交互
