# Proposal: 智能体小镇 经济与金融体系

## Why

小镇目前的"经济"极其单薄:每日结算时给每位活跃居民发放一笔固定随机工资(`WorldSimEngine.java:169`,`60+rnd(120)` 金币/天),随机事件偶尔加减金币,`agent_transaction` 只记录 ≥100 的大额流水。没有消费、没有商业营收、没有小镇整体资金池、没有月度汇总,更没有市场机制。金币只增不减、缺乏循环,居民之间/居民与场所之间没有真实的经济互动,数据在前端也几乎不可见。

本变更把小镇从"发钱"升级为一个**会循环运转的经济体**:居民既挣钱也花钱(在饭店/超市/医院等场所消费),这些场所形成营收,小镇有整体资金总量与统计口径;并在此之上引入**股票市场**,居民可用金币买卖股票产生盈亏,前端把这些经济数据可视化展示出来。

## What Changes

1. **前端样式优化 + 更多数据展示**:重整小镇视图(`#worldView`),新增"经济面板"——展示小镇整体金币总量、今日/本月收支、富豪榜、场所营收、以及居民个人的日/月收入明细。
2. **小镇整体金币(资金池)**:引入小镇级资金总量与统计(所有居民金币之和 + 公共账户),作为经济健康度指标。
3. **居民每日日常消费**:每位居民在每日结算时按作息/需求产生日常花销(吃饭、购物、就医等),从个人金币扣除。消费是经济循环的"支出"侧,与工资/售卖等"收入"侧对冲。
4. **消费与商业营收机制**:让现有地图场所(食肆/杂货铺/回春医馆/集市等)成为"商业体",居民的日常消费按类别流向对应场所,转化为场所营收,形成"挣—花—营收"闭环。
5. **作品售卖收入**:居民创作的作品(`agent_product`:小说/歌/画/短片)按质量转化为售卖收入计入金币与流水,成为收入侧的一个来源(接入既有创作玩法,不改创作本身)。
6. **个人日/月收入**:在现有 `agent_transaction` 流水基础上,聚合出每位居民的每日净收入与每月净收入(收入-支出),持久化 + API 暴露 + 前端展示。
7. **股票市场**:引入若干"小镇股票"(可与场所/行业挂钩),每日随经济状况波动;居民按性格/资产做买卖决策,盈亏计入金币;前端展示行情与居民持仓。

## Scope

### In
- 每日结算(`dailySettlement`)中新增:居民日常消费扣费、场所营收结算、作品售卖收入、月度收入聚合、股价更新、居民炒股决策。
- 新增经济数据的持久化(资金池/场所营收/月度收入/股票与持仓)与只读 API。
- 前端小镇视图的样式优化与经济面板展示。
- 复用现有 `AgentTransaction` 作为统一货币流水底账;复用 `agent_product` 作为售卖收入来源。

### Out
- 不做真实货币/支付,纯虚拟金币。
- 不做玩家(用户)亲自操盘炒股的交互——本期股票由居民 AI 自主决策(用户仅观看);玩家交易可作为后续变更。
- 不引入实时行情/秒级 tick;股价按"每日结算"节奏更新。
- 不重写自主行动/漫游引擎,只在结算钩子里挂经济逻辑。
- 不改动创作/关系/死亡等既有玩法本身(仅让作品产出接入售卖收入)。

## Impact

- **数据库**(`ddl-auto:none`,需手工 ALTER):新增股票行情表、居民持仓表、场所营收表、居民月度收入表;`agent_employee` 可能新增少量冗余汇总列(如 `net_worth` 便于排行,视设计定);`world_settings` 或新表存小镇资金池状态。
- **后端**:`WorldSimEngine` 结算流程扩展(消费/营收/月结/股市);新增经济相关 Service/Entity/Repository;`WorldController` 新增经济只读端点。
- **前端**:`index.html` 小镇视图重构 + 新经济面板 + 图表/榜单渲染。
- **风险**:结算流程变长(消费/股市决策可能触发 LLM 调用→token 成本与限流);数值需调参防止通胀/破产失衡;新表与手工 ALTER 的迁移一致性。

## Capabilities

- `town-economy-core`:小镇整体资金池、居民日常消费扣费、场所营收、作品售卖收入、经济健康统计。
- `resident-income`:居民日/月净收入(收入-支出)聚合与展示(基于统一流水底账)。
- `town-stock-market`:小镇股票行情、居民自主炒股、持仓与盈亏。
- `town-economy-ui`:小镇视图样式优化与经济数据可视化面板。
